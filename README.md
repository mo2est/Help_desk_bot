# 🤖 Help Desk Bot — FAQ-бот для сервисных компаний

Telegram-бот для автоматизации поддержки клиентов. Отвечает на часто задаваемые вопросы, принимает обращения от клиентов и уведомляет администраторов в реальном времени.

**Стек:** Python 3.11+ · aiogram 3 · SQLAlchemy (async) · SQLite · aiosqlite

## Возможности

### Для клиентов
- Меню с категориями FAQ (инлайн-кнопки)
- Мгновенные ответы на популярные вопросы
- Форма отправки произвольного вопроса администратору
- Навигация «Назад» на каждом уровне меню

### Для администраторов
- Команда `/admin` — полная панель управления прямо в Telegram
- Добавление / удаление категорий и вопросов без правки кода
- Подтверждение перед удалением категории (защита от случайной потери данных)
- Просмотр входящих вопросов от клиентов
- Мгновенные уведомления о новых обращениях
- Защита от флуда (throttling): предупреждение один раз, дальше сообщения молча игнорируются

### Надёжность и безопасность
- Доступ к админ-панели контролируется единым middleware — проверка на входе в роутер, а не в каждом хендлере
- Весь пользовательский контент экранируется перед отправкой (защита от поломки HTML-разметки)
- Сервисный слой покрыт автотестами (pytest)

## Запуск на Windows — пошагово с нуля

> Этот раздел написан для тех, кто никогда не запускал Python-программы.
> Следуйте шагам по порядку — каждый шаг описан максимально подробно.

---

### Шаг 1 — Создать бота в Telegram (5 минут)

1. Откройте Telegram и найдите бота **@BotFather** (официальный бот от Telegram).
2. Нажмите **Start** или введите `/start`.
3. Введите команду `/newbot`.
4. BotFather спросит **имя бота** — это отображаемое имя, например: `Поддержка BeautyPro`.
5. Затем спросит **username** (техническое имя, только латиница, должно заканчиваться на `bot`), например: `beautypro_support_bot`.
6. BotFather пришлёт **токен** — длинная строка вида `7123456789:AAFxxxxxxx`. Скопируйте его и сохраните.

> ⚠️ Токен — это пароль от вашего бота. Никому его не передавайте.

---

### Шаг 2 — Узнать свой Telegram ID

Чтобы получать уведомления от бота как администратор, нужен ваш числовой ID.

1. Найдите в Telegram бота **@userinfobot**.
2. Нажмите **Start**.
3. Бот пришлёт сообщение вида: `Id: 123456789`. Это и есть ваш ID — запишите его.

---

### Шаг 3 — Установить Python

1. Откройте браузер и перейдите на сайт **python.org/downloads**
2. Нажмите жёлтую кнопку **Download Python 3.11.x** (или более новую версию).
3. Запустите скачанный установщик.
4. **Важно:** на первом экране установщика поставьте галочку **"Add Python to PATH"** (внизу окна) — без этого ничего не заработает.
5. Нажмите **Install Now** и дождитесь завершения.

Проверка установки:
- Нажмите `Win + R`, введите `cmd`, нажмите Enter.
- В открывшемся чёрном окне (командная строка) введите:
  ```
  python --version
  ```
- Должно появиться что-то вроде `Python 3.11.9`. Если появилось — всё хорошо.

### Шаг 4 — Скачать код бота

**Вариант А — через браузер (проще):**

1. Перейдите на страницу репозитория на GitHub.
2. Нажмите зелёную кнопку **Code → Download ZIP**.
3. Распакуйте архив. Вы получите папку `Help_desk_bot-main` (или похожую).
4. Переименуйте папку в `Help_desk_bot` и переместите, например, на рабочий стол или в `C:\Projects\`.

**Вариант Б — через Git (если установлен):**

1. Откройте командную строку (`Win + R` → `cmd`).
2. Выполните:
   ```
   git clone https://github.com/<аккаунт>/Help_desk_bot.git
   cd Help_desk_bot
   ```

---

### Шаг 5 — Открыть папку в командной строке

1. Откройте папку `Help_desk_bot` в Проводнике Windows.
2. Кликните на адресную строку вверху (где написан путь к папке), она выделится.
3. Введите `cmd` прямо в адресной строке и нажмите Enter.
4. Откроется командная строка уже внутри нужной папки. ✅

---

### Шаг 6 — Создать виртуальное окружение

Виртуальное окружение — это изолированное пространство для зависимостей бота, чтобы не засорять систему. Выполните в командной строке:

```
python -m venv .venv
```

Затем активируйте его:

```
.venv\Scripts\activate
```

Если всё верно, в начале строки появится `(.venv)` — это значит, окружение активно.

> ⚠️ Если Windows не разрешает запуск скриптов, выполните в PowerShell команду:
> `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned`
> и нажмите `Y`. Затем повторите активацию.

---

### Шаг 7 — Установить зависимости

В командной строке с активным `(.venv)` выполните:

```
pip install -r requirements.txt
```

Начнётся загрузка библиотек. Дождитесь надписи `Successfully installed ...`. Это занимает 1–2 минуты.

---

### Шаг 8 — Создать файл настроек

1. В папке `Help_desk_bot` найдите файл `.env.example`.
2. Скопируйте его и переименуйте копию в `.env` (точка в начале — обязательна).

> Если Windows не даёт создать файл без имени, начинающийся с точки:
> В командной строке выполните: `copy .env.example .env`

3. Откройте файл `.env` блокнотом (правая кнопка → Открыть с помощью → Блокнот).
4. Заполните его:

```
BOT_TOKEN=7123456789:AAFxxxxxxx
ADMIN_IDS=123456789
DATABASE_URL=sqlite+aiosqlite:///./helpdesk.db
```

- `BOT_TOKEN` — токен из Шага 1
- `ADMIN_IDS` — ваш ID из Шага 2 (если администраторов несколько — через запятую: `123,456,789`)

5. Сохраните файл (`Ctrl + S`) и закройте.

### Шаг 9 — Запустить бота

В командной строке с активным `(.venv)` выполните:

```
python main.py
```

Вы увидите сообщение вида:
```
2024-01-15 10:00:00 [INFO] Starting Help Desk Bot...
2024-01-15 10:00:00 [INFO] Admin IDs: [123456789]
```

Откройте Telegram, найдите вашего бота по username и нажмите **Start**. Бот должен ответить приветствием! 🎉

### Шаг 10 — Как оставить бота работать постоянно

**Проблема:** если закрыть командную строку — бот остановится.

**Решение 1 — Свернуть, не закрывать** (подходит для теста):
Просто сверните окно командной строки. Пока оно открыто — бот работает.

**Решение 2 — Оставить компьютер включённым с открытой командной строкой.**
Не самый удобный вариант, но работает для демонстрации.

**Решение 3 — Запустить через bat-файл при старте Windows:**

Создайте файл `start_bot.bat` в папке бота со следующим содержимым:

```bat
@echo off
cd /d %~dp0
call .venv\Scripts\activate
python main.py
```

Затем создайте ярлык на этот файл и скопируйте ярлык в папку автозагрузки:
`Win + R` → введите `shell:startup` → нажмите Enter → вставьте ярлык.

> 💡 **Рекомендация для серьёзного использования:** разместите бота на VPS-сервере (см. раздел [Развёртывание на сервере](#развёртывание-на-сервере-production)). Это обеспечит работу 24/7 без необходимости держать включённым личный компьютер. Аренда минимального сервера стоит от 200–300 ₽/месяц.

### Что делать, если что-то пошло не так (Windows)

| Ошибка | Причина | Решение |
|---|---|---|
| `'python' is not recognized` | Python не установлен или не добавлен в PATH | Переустановите Python с галочкой «Add to PATH» |
| `No module named 'aiogram'` | Зависимости не установлены или окружение не активировано | Убедитесь, что в начале строки стоит `(.venv)`, затем `pip install -r requirements.txt` |
| `BOT_TOKEN environment variable is required` | Файл `.env` не создан или пуст | Проверьте, что файл называется `.env` (не `.env.txt`) и содержит токен |
| `Telegram: Unauthorized` | Токен неверный | Скопируйте токен заново из @BotFather |
| `.venv\Scripts\activate` не работает | Политика выполнения скриптов Windows | Выполните в PowerShell: `Set-ExecutionPolicy RemoteSigned -Scope CurrentUser` |

## Быстрый старт

> Для локального запуска (тестирование, разработка).

### 1. Получить токен бота

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте `/newbot`, следуйте инструкциям
3. Скопируйте выданный токен вида `7123456789:AAF...`

### 2. Узнать свой Telegram ID

Откройте [@userinfobot](https://t.me/userinfobot) — он пришлёт ваш числовой ID (например, `123456789`).

### 3. Клонировать и настроить

```bash
git clone https://github.com/<ваш-аккаунт>/Help_desk_bot.git
cd Help_desk_bot

python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate

pip install -r requirements.txt

cp .env.example .env
```

Откройте `.env` и заполните:

```env
BOT_TOKEN=7123456789:AAF...        # токен от BotFather
ADMIN_IDS=123456789                # ваш Telegram ID (через запятую для нескольких)
DATABASE_URL=sqlite+aiosqlite:///./helpdesk.db
```

### 4. Запустить

```bash
python main.py
```

Откройте бота в Telegram → `/start`. При первом запуске база данных заполняется тестовыми данными автоматически.

## Развёртывание на сервере (production)

### Требования к серверу

| Параметр | Минимум |
|---|---|
| ОС | Ubuntu 22.04 / Debian 12 |
| CPU | 1 ядро |
| RAM | 256 МБ |
| Диск | 1 ГБ |
| Python | 3.11+ |
| Доступ | SSH, открытый исходящий HTTPS (443) |

---

### Шаг 1 — Подготовка сервера

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y python3.11 python3.11-venv python3-pip git
```

### Шаг 2 — Создание системного пользователя

```bash
sudo useradd -m -s /bin/bash botuser
sudo su - botuser
```

### Шаг 3 — Получение кода

```bash
git clone https://github.com/<ваш-аккаунт>/Help_desk_bot.git
cd Help_desk_bot

python3.11 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### Шаг 4 — Конфигурация окружения

```bash
cp .env.example .env
nano .env
```

```env
BOT_TOKEN=7123456789:AAF...
ADMIN_IDS=123456789,987654321
DATABASE_URL=sqlite+aiosqlite:////home/botuser/Help_desk_bot/helpdesk.db
```

Ограничьте права на файл с секретами:

```bash
chmod 600 .env
```

### Шаг 5 — Создание systemd-службы

Вернитесь в root-сессию:

```bash
exit   # выход из botuser
sudo nano /etc/systemd/system/helpdesk-bot.service
```

Вставьте:

```ini
[Unit]
Description=Telegram Help Desk Bot
After=network.target

[Service]
Type=simple
User=botuser
WorkingDirectory=/home/botuser/Help_desk_bot
EnvironmentFile=/home/botuser/Help_desk_bot/.env
ExecStart=/home/botuser/Help_desk_bot/.venv/bin/python main.py
Restart=always
RestartSec=5
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

Активируйте и запустите:

```bash
sudo systemctl daemon-reload
sudo systemctl enable helpdesk-bot
sudo systemctl start helpdesk-bot
```

Проверьте статус:

```bash
sudo systemctl status helpdesk-bot
```

Должно отобразиться `active (running)`.

### Шаг 6 — Просмотр логов

```bash
# Все логи в реальном времени
sudo journalctl -u helpdesk-bot -f

# Логи за сегодня
sudo journalctl -u helpdesk-bot --since today
```

```env
DATABASE_URL=sqlite+aiosqlite:////app/data/helpdesk.db
```

Запуск:

```bash
docker compose up -d
docker compose logs -f
```

## Конфигурация

Все настройки задаются через переменные окружения в файле `.env`.

| Переменная | Обязательная | Описание |
|---|---|---|
| `BOT_TOKEN` | ✅ | Токен бота от @BotFather |
| `ADMIN_IDS` | ✅ | Telegram ID администраторов через запятую |

```env

```

Все перечисленные пользователи получат доступ к `/admin` и уведомления о новых вопросах.

## Сценарии использования

### Клиентский путь

```
/start
  └─► Главное меню (категории)
        └─► Список вопросов категории
              ├─► Ответ на вопрос
              │     ├─► [Назад к вопросам]
              │     ├─► [Главное меню]
              │     └─► [Задать свой вопрос]
              └─► [Задать свой вопрос]
                    └─► Ввод текста → отправка → уведомление администратору
```

### Административный путь

```
/admin
  ├─► Категории
  │     ├─► [Категория] → Список вопросов
  │     │       ├─► [Вопрос] → Просмотр / Удалить
  │     │       ├─► [Добавить вопрос] → FSM: текст → ответ
  │     │       └─► [Удалить категорию] → Подтверждение → Удаление
  │     └─► [Добавить категорию] → FSM: название → эмодзи
  │
  └─► Вопросы пользователей
        └─► [Вопрос #N] → Просмотр → [Отметить отвеченным]
```

## Тесты

Сервисный слой и контроль доступа покрыты автотестами (pytest, in-memory SQLite — реальная база не затрагивается):

```bash
# Установить тестовые зависимости
pip install pytest pytest-asyncio

# Запустить все тесты
python -m pytest
```

Что проверяется:
- CRUD категорий и вопросов, включая каскадное удаление вопросов вместе с категорией
- Валидация: вопрос нельзя создать в несуществующей категории
- Полный цикл обращения пользователя: сохранение → список неотвеченных → отметка «отвечено»
- Идемпотентность демо-данных: повторный запуск не дублирует контент
- Контроль доступа: админ проходит, посторонний получает отказ, пустой список админов блокирует всех

## Настройка контента

### Через административную панель (рекомендуется)

1. Откройте бота в Telegram
2. Введите `/admin`
3. Выберите **Категории → Добавить категорию**
4. Укажите название и эмодзи
5. Войдите в созданную категорию и добавляйте вопросы через **Добавить вопрос**

### Замена демо-данных

Тестовые данные загружаются только при первом запуске (если база пуста). Чтобы сбросить их и начать заново:

```bash
# Остановить бота
sudo systemctl stop helpdesk-bot

# Удалить базу данных
rm helpdesk.db

# Запустить снова — данные пересоздадутся
sudo systemctl start helpdesk-bot
```

После этого отредактируйте содержимое через `/admin` или напрямую измените `app/seed.py` перед первым запуском.

## Частые проблемы

### Бот не отвечает сразу после запуска

Telegram иногда задерживает доставку первого сообщения на 5–10 секунд. Подождите и повторите `/start`.

### `BOT_TOKEN environment variable is required`

Файл `.env` не найден или пуст. Убедитесь, что он находится в директории запуска бота и содержит корректный `BOT_TOKEN`.

### `PermissionError` при создании базы данных

Пользователь, от которого запущен бот, не имеет прав на запись в директорию. Проверьте:

```bash
ls -la /home/botuser/Help_desk_bot/
```

И при необходимости исправьте:

```bash
sudo chown -R botuser:botuser /home/botuser/Help_desk_bot/
```

### Команда `/admin` не работает

Ваш Telegram ID не добавлен в `ADMIN_IDS`. Узнайте ID через [@userinfobot](https://t.me/userinfobot) и добавьте в `.env`, затем перезапустите бота:

```bash
sudo systemctl restart helpdesk-bot
```

### Бот перестал отвечать после долгой работы

Проверьте логи на наличие ошибок сети:

```bash
sudo journalctl -u helpdesk-bot -n 50
```

Перезапустите службу:

```bash
sudo systemctl restart helpdesk-bot
```

Служба настроена на автоматический перезапуск (`Restart=always`), поэтому в большинстве случаев восстановление происходит автоматически.
